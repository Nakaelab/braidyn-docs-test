User Guide

Accessing BraiDyn-BC data with CLI and AWS EC2

This page explains how to access the BraiDyn-BC data from your local machine using AWS CLI, and how to work with the same data directly from an AWS EC2 instance.

Overview

There are two main ways to work with the dataset:

Local machine + AWS CLI

Suitable when you want to browse, test, or download data to your own computer.

AWS EC2 + AWS CLI

Suitable when the dataset is large and you want to process it directly on AWS without storing everything on your local machine.

Method 1

Download from your local machine using AWS CLI

1

Check that AWS CLI is installed

Open Terminal or PowerShell and run:

aws --version
2

List the bucket contents

Use this command to confirm that you can access the dataset:

aws s3 ls s3://braidyn-bc-buckets --no-sign-request
3

Download all data

This command downloads the contents of the bucket into a local data folder:

aws s3 sync s3://braidyn-bc-buckets ./data/ --no-sign-request
4

Download a specific folder only

If you only need a subset, specify the path:

aws s3 sync "s3://braidyn-bc-buckets/sub-VG1-GC#105" "./data/sub-VG1-GC#105" --no-sign-request

Method 2

Access the data directly from AWS EC2

This option is useful when the dataset is large and you want to analyze it on AWS instead of downloading it to your personal computer.

1

Connect to your EC2 instance

Start an EC2 instance and connect to it using SSH or EC2 Instance Connect.

2

Confirm AWS CLI is available

aws --version
3

List files in the S3 bucket

aws s3 ls s3://braidyn-bc-buckets --no-sign-request
4

Download data to the EC2 instance

aws s3 sync s3://braidyn-bc-buckets ./data/ --no-sign-request
5

Download only a specific folder on EC2

aws s3 sync "s3://braidyn-bc-buckets/sub-VG1-GC#105" "./data/sub-VG1-GC#105" --no-sign-request

Notes

Storage

Make sure your local machine or EC2 instance has enough disk space before starting a large download.

AWS costs

Downloading on your own computer does not require EC2, but using an EC2 instance may incur AWS charges.

Stopping downloads

You can stop a running download with Ctrl + C and resume later with the same aws s3 sync command.

EC2 shutdown

If you use EC2, stop the instance after you are done to avoid unnecessary charges.
